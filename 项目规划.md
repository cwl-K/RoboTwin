# GAPA 项目规划

**项目名称：**  
**GAPA: Language-Guided Grasp-Anything-and-Put-Anywhere Manipulation in RoboTwin2.0 with Vision-Language Self-Correction**

**中文名称：**  
**GAPA：基于 RoboTwin2.0 的语言引导任意抓取与任意放置系统及视觉语言自纠错**



---

## 1. 项目背景

课程题目要求构建一个 **“grasping-anything-to-put-anywhere robot system”**，即机器人能够根据用户输入的自然语言任务，在 RoboTwin2.0 仿真环境中完成抓取和放置类操作。

官方 RoboTwin 的 Expert Code Gen 提供了一条重要思路：

```text
任务描述 + actor_list + API 示例
→ LLM 生成 play_once()
→ 仿真执行
→ 错误反馈 / 多模态反馈
→ 重新生成
→ 保存 expert code
```

但官方流程主要目标是为已有任务生成可运行专家代码，而不是面向用户当前场景做实时自然语言交互、视觉感知执行和纠错视频展示。

因此，本项目在官方 Expert Code Gen 的基础上，进一步扩展为一个完整的自然语言机器人任务执行系统：

```text
自然语言任务输入
→ 任务解析与可行性判断
→ 候选代码 / SkillPlan 生成
→ 多随机场景直接读取物体的位姿，进行验证
→ 当前前端场景使用VLM感知执行
→ 失败检测与视觉语言反馈
→ 参数修正 / 感知重估 / 策略切换 / 代码再生成
→ 输出带纠错过程的演示视频
```

---

## 2. 项目目标

本项目最终希望实现一个基于 RoboTwin2.0 的 GAPA 系统，使用户可以在前端界面中：

1. 点击按钮生成一个随机仿真场景；
2. 在输入框中输入自然语言任务；
3. 系统判断任务是否可以完成；
4. 系统生成多个候选 SkillPlan / 代码；
5. 系统在多个随机验证场景中直接读取物体的位姿，筛选最优计划；
6. 系统回到用户当前前端场景，使用 VLM 感知结果执行任务；
7. 如果执行失败，系统通过错误文本和视觉语言反馈生成 failure_report；
8. 系统根据 failure_report 决定修参数、重估感知、换策略或重生成代码；
9. 系统保存每次尝试、失败、诊断、修改和重试过程；
10. 最终生成一个带纠错过程的实验视频。

---

## 3. 核心思想

GAPA 的核心不是让 LLM 直接输出底层关节角或连续动作，而是采用分层结构：

```text
自然语言
→ TaskDSL
→ SkillPlan
→ SkillLibrary
→ RoboTwin 高层 API
→ 分步执行与验证
→ 失败反馈与纠错
```

其中最重要的设计原则是：

### 3.1 LLM 负责高级任务规划，不直接控制底层动作

LLM 不应该直接输出：

```python
move_to([0.31, -0.08, 0.76, ...])
```

而应该输出结构化技能计划：

```json
[
  {"skill": "grasp_object", "object_name": "apple"},
  {"skill": "place_in", "object_name": "apple", "target_name": "basket"}
]
```

底层动作由 SkillLibrary 封装 RoboTwin API 完成。

### 3.2 Oracle 只用于离线候选筛选，不用于当前前端场景最终执行

为了避免“作弊”嫌疑，系统必须区分：

```text
离线验证阶段：直接读取物体的位姿
前端最终执行阶段：必须使用VLM感知
```

### 3.3 最终视频要保留纠错过程，而不是只展示成功结果

本项目的一个重要展示亮点是：

```text
第一次抓取失败
→ 视觉语言模型分析失败原因
→ 系统调整抓取点 / 参数
→ 第二次抓取成功
→ 继续完成后续任务
```
